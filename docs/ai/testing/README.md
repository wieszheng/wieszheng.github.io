# AI 测试开发学习指南

> 适用人群：有测试开发基础，想转型 AI 测试方向的工程师

---

## 为什么选择 AI 测试开发？

### 行业趋势

- AI 应用爆发式增长，大模型、RAG、Agent 等应用场景快速落地
- 传统测试方法论无法完全覆盖 AI 应用的测试需求
- 懂 AI 测试的人才稀缺，薪资水平较高

### 发展前景

- **测开天花板突破**：从传统自动化测试走向 AI 测试专家
- **技能可迁移**：LLM 评测、RAG 测试、Agent 测试等能力通用性强
- **跨领域机会**：可向 AI 产品、AI 研发等方向延伸

## AI 应用测试概览

### RAG 测试（检索增强生成）

RAG 将**检索**与**生成**结合：
1. 用户提问 → 2. 检索相关文档 → 3. 文档作为上下文 → 4. LLM 生成回答

| 维度 | 说明 | 评测指标 |
|------|------|----------|
| 检索质量 | 能否找到正确的文档 | Recall@K、MRR、NDCG |
| 生成质量 | 回答是否准确、流畅 | 准确率、相关性评分 |
| 幻觉检测 | 是否编造不存在的信息 | 幻觉率、事实一致性 |
| 端到端性能 | 整体响应时间和质量 | 延迟、吞吐量、用户满意度 |

### Agent 测试（智能体）

Agent 是能够自主规划、调用工具、执行任务的 AI 系统：

| 维度 | 说明 | 测试要点 |
|------|------|----------|
| 工具调用 | 该调 API 时是否正确调用 | 调用时机、参数正确性 |
| 多轮对话 | 上下文记忆是否准确 | 信息保持、话题切换 |
| 规划能力 | 任务拆解是否合理 | 步骤完整性、执行顺序 |
| 错误处理 | 异常情况是否恢复 | 重试、降级、提示用户 |
| 安全性 | 是否执行危险操作 | 越权、注入、敏感信息 |

### 推荐工具

## 学习路线图

### 第一阶段：基础入门（1-2 周）

- 了解 Transformer 原理
- 学习 Prompt Engineering 基础
- 体验主流 LLM：ChatGPT、Claude、Kimi、通义千问
- 理解 LLM 的能力和局限

### 第二阶段：工具实践（2-4 周）

- 学习使用 LangChain / LlamaIndex
- 搭建一个简单的 RAG 应用
- 使用 Ragas 进行评测
- 尝试 Promptfoo 进行 prompt 测试

### 第三阶段：深入专项（1-2 个月）

- 深入学习 RAG 测试方法论
- 学习 Agent 测试框架
- 了解多模态测试
- 参与开源项目或自建项目

### 第四阶段：项目实战（持续）

- 在公司项目中应用 AI 测试
- 总结最佳实践，输出技术文章
- 持续关注行业动态

## 常见问题

**Q：没有 AI 项目经验怎么办？**
A：自己搭建项目是最好的方式。选择一个小场景，从零开始实现，过程中会遇到真实问题。

**Q：需要学深度学习吗？**
A：不需要深入，但需要理解基本原理。重点是会用、会测，而不是会训练。

**Q：如何证明自己的能力？**
A：GitHub 开源项目 + 技术博客输出 + 实际项目成果。

## 总结

AI 测试开发的核心要点：

1. **打好基础**：传统测试能力 + AI 知识
2. **动手实践**：做项目是最好的学习
3. **持续学习**：这个领域变化很快，保持学习
4. **输出沉淀**：写文章、做分享，加深理解

> **最重要的一点**：不要只看不练，看 100 篇文章不如自己动手做一个项目。
